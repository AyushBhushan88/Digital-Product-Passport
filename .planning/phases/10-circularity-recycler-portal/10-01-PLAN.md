# Plan: 10-01 - Circularity & Recycler Portal

## Goal
Implement the core functionality for the Recycler Portal, including authenticated access to disassembly guides and on-chain status updates.

## Strategy
1. **Recycler Authentication**: Implement a simple API key or role-based check for the Recycler endpoints.
2. **Blockchain State Transitions**: Extend `BlockchainService` to support updating the status of a Digital Twin (e.g., MINTED -> RECYCLED).
3. **Recycler API**: Create a `RecyclerController` to handle disassembly data retrieval and status updates.
4. **Data Privacy**: Ensure sensitive disassembly instructions are only accessible via the Recycler endpoints.

## Detailed Tasks

### 1. Blockchain Service Extension
- [ ] Add `updateTwinStatus(tokenId: string, newStatus: string): Promise<string>` to `BlockchainService`.
- [ ] Implement status transition validation logic.

### 2. Recycler Module & Controller
